Method to determine the appropriate mode and version for a QR code.

Method accepts three parameters, and determines the appropriate version given a desired version and error correction level. If the given version is too small to hold the data, an appropriate one is selected.

Parameters
versionInteger between 1 and 40 that specifies the desired size of the QR code.
eclAccepts one of four values: 'L', 'M', 'Q', 'H' that specify the level of error correction to be encoded.
inputStringAccepts an input string to be encoded into data bits for a QR code.
Returns
Appropriate version

Method to obtain data bits for a QR code.

Method accepts four parameters.

Parameters
versionInteger between 1 and 40 that specifies the size of the QR code.
eclAccepts one of four values: 'L', 'M', 'Q', 'H' that specify the level of error correction to be encoded.
modeAccepts one of four values: NUMERIC, ALPHANUMERIC, BYTE, KANJI that specify the acceptable characters and encoding mode for the QR code.
inputStringAccepts an input string to be encoded into data bits for a QR code.
Returns
Encoded data bits. Method accepts four parameters version: QR code version ecl: QR code error correction level mode: QR code encoding mode inputString: String to be encoded.
